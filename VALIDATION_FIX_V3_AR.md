# تقرير التحقق — V3

تم التحقق من التالي داخل بيئة التسليم:

- فتح ملف المشروع وفحص البنية كاملة.
- فحص صياغة جميع ملفات Dart وعددها 26 بواسطة Dart tree-sitter parser: ناجح.
- فحص صياغة `MainActivity.kt` باستخدام Kotlin compiler مع stubs: ناجح.
- فحص جميع ملفات Android XML: ناجح.
- فحص ملفات YAML الأساسية: ناجح.
- التحقق المنطقي من حالات توحيد أرقام الهاتف المطلوبة.
- التحقق من سلامة ملف ZIP النهائي بعد إنشائه.

لم يتم تنفيذ بناء Flutter أو `flutter analyze` في بيئة التسليم لأن Flutter SDK غير مثبت. يلزم على جهاز التطوير تشغيل:

```bash
flutter clean
flutter pub get
flutter analyze
flutter test
flutter run
```

## اختبار يدوي مهم

1. أضف مفتاح Google Maps إلى `android/local.properties`.
2. أوقف التطبيق بالكامل وشغله من جديد.
3. افتح الخريطة وتأكد من ظهور البلاطات والنقطة الزرقاء.
4. تحرك عدة أمتار وتأكد أن الموقع يتحدث.
5. جرّب رقمًا محليًا يبدأ بـ `05` من زر واتساب.
6. راقب اسم المتجر في بطاقة المهمة وMarker الخريطة.

إذا بقي اسم متجر محدد غير ظاهر، فالاستجابة الحقيقية لذلك المتجر تستخدم حقلًا غير موجود في العينة المتاحة؛ افتح شاشة تشخيص `/tasks` وشارك فقط أسماء الحقول بعد إخفاء التوكن والبيانات الحساسة.

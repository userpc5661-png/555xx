# SLS Assistant Pro

تطبيق Flutter مساعد للمندوب، يحافظ على تسجيل الدخول وواجهات SLS الحالية، ويضيف واجهة يومية أوضح للمهام والخريطة والمسح.

## ما تم تنفيذه في هذا الإصدار

- إصلاح قراءة رقم الشحنة، مع دعم الحقل الرسمي `order_awb` وحقول بديلة.
- إصلاح قراءة الإحداثيات، مع دعم `delivery_location_lat` و`delivery_location_lng` وحقول الإحداثيات المركبة.
- فتح خرائط Google بالإحداثيات، أو بالعنوان كحل احتياطي عندما لا تتوفر الإحداثيات.
- دعم ردود `/tasks` التي تحتوي على `tasks`، وكذلك المهام التي تجمع عدة `orders`.
- إضافة تنقل سفلي بأربع صفحات:
  - Dashboard
  - Tasks
  - Map
  - Scanner
- إضافة إحصائيات اليوم: الإجمالي، المتبقي، المنجز، الملغي، COD، والمدفوع مسبقًا.
- إضافة البحث وفلاتر طريقة الدفع والحالة.
- إضافة خريطة OpenStreetMap تعرض جميع الشحنات ذات الإحداثيات كعلامات قابلة للضغط.
- إضافة الوضع الداكن والتبديل السريع من شريط التطبيق.
- إضافة صلاحيات الكاميرا والموقع المطلوبة لنسخة iOS.
- إخفاء `api_token` من رابط صفحة التشخيص.

## التشغيل

بعد فك الضغط، افتح المشروع في Android Studio أو VS Code ثم شغّل:

```bash
flutter pub get
flutter test
flutter run
```

> تمت إضافة حزمتَي `flutter_map` و`latlong2`، لذلك يجب تشغيل `flutter pub get` قبل البناء.

## ملاحظات مهمة

- التطبيق لا يخزن كلمة المرور؛ يتم حفظ بيانات الجلسة في التخزين الآمن فقط.
- تصنيف COD يعتمد على `is_cod` و`cod_amount` وحقول الدفع المتوفرة في رد السيرفر.
- عدادات المنجز والملغي تعتمد على كود/اسم الحالة الموجود في `/tasks`.
- إذا كانت شحنة بلا إحداثيات ولكن لديها عنوان، يبقى زر الموقع فعالًا ويفتح البحث بالعنوان.
- يلزم اختبار ميداني بحساب حقيقي للتأكد من أسماء الحالات والقيم التي يعيدها السيرفر لكل نوع مهمة.


## تحديث الخريطة المباشرة

تحتوي هذه النسخة على Google Maps داخل التطبيق مع تتبع مباشر لموقع المندوب، وعلامات لجميع العملاء الذين لديهم إحداثيات، وإظهار اسم المتجر ورقم الشحنة والهاتف والموقع في التفاصيل. قبل تشغيل صفحة الخريطة، طبّق الخطوات الموجودة في `GOOGLE_MAPS_SETUP_AR.md` لإضافة مفتاح Google Maps الخاص بك.

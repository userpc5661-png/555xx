# إعداد Google Maps — خطوة مطلوبة مرة واحدة

الخريطة داخل التطبيق تستخدم Google Maps رسميًا، لذلك تحتاج مفتاح Android خاص بمشروعك. لا يمكن وضع مفتاح عام داخل ملف ZIP.

## 1) Google Cloud

- فعّل **Maps SDK for Android**.
- فعّل **Billing** للمشروع.
- أنشئ API Key وقيّده إلى **Android apps**.
- استخدم اسم الحزمة الذي يظهر داخل شاشة الخطأ في التطبيق. الاسم الافتراضي حاليًا:

```text
com.example.sls_assistant_pro
```

- أضف SHA-1 الخاص بتوقيع Debug أو Release المستخدم في البناء.

## 2) Android Studio

افتح الملف:

```text
android/local.properties
```

لا تحذف سطر `sdk.dir`. أضف تحته:

```properties
GOOGLE_MAPS_API_KEY=ضع_مفتاح_Android_الحقيقي_هنا
```

ثم نفّذ:

```bash
flutter clean
flutter pub get
flutter run
```

## 3) إذا بقيت الخريطة فارغة

- تأكد أن المفتاح ليس مفتاح iOS أو Web.
- تأكد أن Package Name وSHA-1 مطابقان للبناء الحالي.
- تأكد من تفعيل Maps SDK for Android وBilling.
- أوقف التطبيق بالكامل ثم شغله من جديد؛ لا تعتمد على Hot Restart بعد تغيير المفتاح.

## الوظائف الموجودة

- النقطة الزرقاء لموقع المندوب.
- تحديث الموقع الحي أثناء الحركة.
- زر إعادة التمركز والمتابعة.
- علامات جميع العملاء ذوي الإحداثيات.
- بطاقة العميل تعرض اسم المتجر، رقم الشحنة، الهاتف، العنوان، والدفع.
